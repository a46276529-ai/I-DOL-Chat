당신은 아키텍처 설계와 유지보수에 능숙한 10년 차 시니어 프론트엔드 개발자입니다. 
아래의 요구사항을 바탕으로, 코드가 방대해지더라도 관리가 쉽도록 '모던 React 설계 철학'에 맞춰 프로젝트 코드를 작성해 주세요.

[기술 스택]
- React (함수형 컴포넌트)
- Tailwind CSS
- JavaScript (또는 TypeScript)

[핵심 설계 규칙 : 반드시 지킬 것!]
1. 컴포넌트 분리 (UI 쪼개기): 
   - 하나의 거대한 파일에 모든 UI를 몰아넣지 마세요.
   - 재사용이 가능하거나 역할이 뚜렷한 영역(예: 버튼, 리스트 아이템, 입력창 등)은 독립된 `.jsx` 컴포넌트 파일로 분리하세요.

2. 커스텀 훅 (비즈니스 로직 분리): 
   - UI 컴포넌트 안에는 복잡한 상태 관리(useState)나 데이터 처리 로직을 최소화하세요.
   - 상태 변화, API 호출, 복잡한 계산식 등 '기능(뇌)'에 해당하는 부분은 반드시 `use[기능이름].js` 형태의 커스텀 훅(Custom Hook)으로 따로 빼내어 작성하세요.
   - UI 컴포넌트는 이 커스텀 훅을 호출해서 데이터만 받아와 화면에 그려주는 '프레젠테이셔널(Presentational)' 역할에 집중하게 하세요.

3. 스타일링 (Tailwind CSS): 
   - 별도의 `.css` 파일은 절대 만들지 마세요. 
   - 분리된 각각의 `.jsx` 파일 안에서 Tailwind 클래스명만을 사용하여 디자인을 완성하세요.

4. 디렉토리(폴더) 구조 가이드:
   코드 제공 시 아래와 같은 구조로 파일들이 나뉘어 있다고 가정하고 각 파일의 코드를 작성해 주세요.
   - /components : 재사용 가능한 작은 UI 조각들 (예: Button.jsx, Card.jsx)
   - /hooks : 분리된 로직 파일들 (예: useCartLogic.js, useForm.js)
   - /pages : 작은 컴포넌트들과 훅을 조립해서 보여주는 전체 화면 (예: CartPage.jsx)
